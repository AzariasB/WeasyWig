    Architecture

    The editor have an MVC architectural pattern. MVC stands for Model-view-controller. The model is the data-storage part. The view correspond to the display and the users interaction. And the controller is working with both the model and the view to display the good informations and modify it when needed.

    • The controller

      There is a main controller for the editor. This controller is called 'Weasywig', you'll find the main function of the editor on it

    • The View

      There is a view. The view is simply called 'view.js'. In the view, there are all the events listeners and the function to interact with the DOM object If you want to show a modal, hide a div or interact with the DOM, you have to pass by the view. Either you can add a function in the views prototype or you can call a function that return a DOM object and then interact with this object.

    • The Model

      There is not a unique model object. The model is composed with the differents '.js' files containing informations for the controller Then, this data is giver to the controller, who can manipulate it, and/or give access to the view.

    Here's a sketch to help you undrestand how the editor architectural pattern is working.

    Librairies

    To help you improve the editor, some librarires are at your disposal. The main library is the easyModal one. Others are not really librairies, it's more plugins. Like the contextmenu and the states manager ones.
    The easyModal library is a set of function to easyly create usefull modals.
    The contextmenu plugin, is to customize the contextmenu wich is displayed when the user righ-cick on an element on the editor
    The states manager plugin let you create accessible states to navigate to when you're editing your page.

    Dependencies

    The project require some basics css and js librairies. Of course, these libraires allready are in the project.

    • First of all, and the main dependency is the jQuery library. The minimum version must be 1.9
    • To create some nice visual features, jQuery UI is a very usefull js/css library which is used in this project.
    • Then to have a good css base, the well-known css framework bootstrap is necessary. The minimum used version must be 3.0
    • To be able to do basic data-processing, the editor is using underscore.js this library offers a lot of usefull functions, and the documentation is really well-made. You can check the existing function if you need some of thems during the creation of modules
    • You've certainly saw that it's possible to use keyboard shorcuts in the editor. The handle of these shortcuts is made thanks to mousetrap this is a great library to handle keyboard inputs.

    Make sure to have all those library if the project is not working. If you're sure you've got them all, be carefull about the versions you're using. And if its still not working, even if you have the good versions, please contact us (see homepage)
