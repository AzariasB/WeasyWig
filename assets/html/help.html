General informations

This editor is a Wysiwyg editor. Wysiwyg stands for What You See is What You Get There are several ways to use Wysiwyg. One way is to drag'n'drop the elements in divs. We decided that it would be more secure and easier to have a visual of where you can add your content. The places where you can add what you want have a big 'plus' in the middle. And when you hover your mouse on these sections, the cross will animate. Now if you click on this div, a sideBar will open to propose you the things you can add it this div. The possible choices are separate into three parts : the basic css components (text,heading), the more complex components, and finally, the javascrip-requiring components. Of course you don't need to have any knowledge in code. It's just a way to organise the features. If you're getting lost in all these, you can search the feature you want. If the feature you're looking for exists, the accordion will unfold, and the feature you're looking for will look like this

Nobody's perfect, so if you missclick on a bad div and you didn't want that to happen, you can click on 'nevermind' and it will cancel the adding action.

Shortctus

To save your times, we set some shortcuts so that you just have to do a combination of keys on you keyboard. The major part of the existing shortcuts are indicated with a little tag Like this Here is the list all of the available shortcuts :

  • Ctrl + s

    Save your project (see description down below)

  • Ctrl + z

    To undo the last action you did (see description down below)

  • Ctrl + right

    Go to the next view. (see descrption down below)

  • Ctrl + left

    Go to the previous view (see decription down below)

  • Ctrl + r

    Restart you project (see description down below)

  • Ctrl + h

    Go on the home page (see description down below)

Options

  • Save

    Create a project is a good thing. But if you can save it, it's even better. So by clicking this button, you can save your project, leave your work, close the tab. And then when you come back, the project will be proposed in the main page. If you didt' give any name to your project yet, one name will be asked to you.

    Attention, if you give a name to your project, and this project name already exists, the existing one will be replaced by the new one.

    So you can create multiple projects and name them differently.

  • Undo

    If you missclick and place a new element at a bad place, you can cancel your action. By clicking on this button, or by doing the shortcut : ctrl + z. The last block you placed will be removed. You can undo straight multiple actions. But if you want to restart the whole project, the best way is to click on 'restart'.

    The undo action only work for the 'add' action, not for the actions such as 'changing font color' and those proposed by the contextmenu

  • Restart

    Sometimes, you can just think 'well, my project is not that good, I'd better restart from the beginning'. In this case, you just have to click on the 'restar' button. Be really carefull : this action is no revesrible If you're not sure if you want to keep your work, it would be better to create a whole new project instead of delete the current and lose all the time spent.

    Once again, if you restart a project, you can't undo the restart action, be really carefull

  • Documentation

    This part is mostyl for the developpers who want to develop plugins for the weasywig editor. The documentation is composed of the differents API of the Weasywigs components. If you're not a developper, you'll get lost, but it can helps you understanding the all working of the editor.

  • Home

    We don't want you to get stuck on the editor, so you can go back on the main page. Just click this button and go back to the menu

    If you don't save your project when leaving, it will not save automatically. So don't forget to save before leaving.

Contextmenu options

You can access to these options at every moment, just by righ-clicking on a created div. The contextmenu will only appear if you clicked on a created div. It won't work on the available boxes. (the ones with the '+' in it)

  • Change font color

    To add some colors to your creation, you can change the fonts color of the selected div. You can enter differents values for color :

    • An hexadecimal value (looking like this : #123123 )
    • A red-green-blue value. Someting like this :rgb(1,2,3)
    • An existing color in html colors : existing values

  • Change background color

    Because changing the font color is not enougth, you can also change the background-color of a div. The possibles colors are the same as the font color.

  • Change font size

    The default font-size might be to small for you (or too big ?). So you can change it in two clicks If you enter a simple number, the default unit will be the pixels, but you can change this just by adding the good unit of mesure right beind the value. Like this 1.3em If you give a unit it must be a css-valid unit.

  • Remove

    If you want to remove a part of your page, you can, just by choosing this option

    The remove action cannot be cancelled

Differents Views and modes

You can go from a view to another by choosing it on the right dropdown of the navbar. An other way to change is simply to do the sorctus ctrl + left or ctrl + rigt . A piece of take ! You can change of view as many times as you want.

  • Editor

    The main part of the editor : the editor itself. Where you can change remove, add blocks to your page. It is the default page of the editor.

  • View result

    If you want to have a preview of what your page is giving without all the available divs. Thanks to this view, you can clearly see how your page look, and if it actually look as you expected. If it does no suits to you, you can come back to the editor and change what you have to change.

  • Edit code

    You can see the HTML code generated by the editor. You can either copy-paste it or edit it. We're sorry for the bad display. Weasywig did not succeed to display it as well as it should be and because of that, it's really hard to edit the code directly in the app. We recommand to copy-paste it into your favorite editor, modify it, and then paste the new code into the editable div. As soon as you come back to the editor, the displayed content correspond to the code wich was in the div.
